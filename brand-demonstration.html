Future Rebranding Considerations

  1. All logos are centralized in src/assets/ (UI) and public/ (docs) — a single file swap rebrands the entire site
  2. The Logo component in components/assets/index.tsx accepts a color prop — this controls the SVG filter, not the file itself
  3. The Xilos brand SVGs have 3 color variants (blue, black, white) — swap the import to change the site-wide logo color
  4. Keep the original brand asset files in ~/Pictures/MPR/product-logos/ as the source of truth
  5. If renaming the product, update: Logo.svg, Logo-2.svg, favicon, theme.config.tsx, and the <title> in index.html
